# AGENT.md — CBC-Stratgems 代码调研任务

## 项目目标

根据以下 HELLDIVERS 2 战备呼叫系统的设想，在 CBC (CreateBigCannons) 源码中找到对应的实现路径，并整理出一份技术设计文档。

**CBC 源码仓库：** https://github.com/Cannoneers-of-Create/CreateBigCannons (MIT, NeoForge 1.21.1)

## 设想内容（结构化版本）

### 核心概念

将 HELLDIVERS 2 的战略配备呼叫系统引入 Minecraft，基于 CBC 的火炮系统实现火力打击。

### 物品与流程

#### 核心物品

- **战略配备呼叫装置** — 主动道具，手持时长按右键打开战备面板（松开关闭并取消）

#### 操作流程

1. 手持呼叫装置 → 长按右键 → 打开战备面板
2. 使用 WASD 输入战备指令（无前缀重复）
3. 输入正确 → 装置变为 **战略配备信标**
4. 右键投掷信标（效果类似雪球/末影珍珠，非消耗品数量，而是变回呼叫装置）
5. 信标锁定对应战备指令并进入 CD

#### 信标落地后

- 在落点生成原版红色信标光柱（后期版本可能改为其他实现）
- 光柱上显示：战略配备名 + icon + 抵达倒计时（布局同 HELLDIVERS 2）
- 显示方式可利用原版方法或自行实现
- 倒计时结束后，利用 CBC 火炮系统实现火力打击（从高处发射预设炮弹，类型/次数/散布等根据战备而定）

### 战备内容

#### 内置战备

- 模组自带提供若干 HELLDIVERS 2 中已有的战略配备

#### 创造模式自定义

- 玩家可在创造模式下自定义战备：
  - 自定义指令（不能前缀重复）
  - 炮火实现方式
  - 名称
  - icon（需自行准备材质）

### 解锁机制

- 添加 **战略配备许可证** 物品
- 玩家持有战略配备信标时使用该许可证，将对应战备录入以供使用
- 创造模式玩家自定义的战备也有对应的许可证

### 兼容性

- 不仅兼容 CBC 本体火炮系统，对 CBC 附属实现的炮弹机制应具有自动兼容性
- 设计上要足够鲁棒，尽量不去触及容易引发兼容问题的方法

### 版本计划

- 优先支持：NeoForge 1.21.1
- 后续支持：Forge 1.20.1

### 未来扩展方向（非火力打击）

以下为后续可能加入的机制，当前不纳入优先实现范围，但要求代码架构上具备高度模块化和可扩展性：

- **支援炮台** — 信标落点放置方块或实体，或预先保存的方块结构
- **装备补给** — 信标落点放置容器（箱子或自行实现的可展示 TACZ 枪械的容器）

## 调研任务

请在 CBC 源码中逐一调研以下模块，找到对应的类、方法、数据结构，并记录：

### 1. 炮弹生成与发射系统

**目标：** 找到从空中生成并发射炮弹的完整链路。

需要回答：
- CBC 中炮弹实体是什么类？如何实例化？
- 发射/生成炮弹需要哪些参数（类型、坐标、朝向、速度、伤害等）？
- 是否有现成的 API 可以在指定位置生成并发射炮弹（而非通过炮管方块）？
- CBC 的炮弹类型有哪些？是否可通过代码注册自定义炮弹？
- 散布/随机偏移机制在哪里实现？

### 2. 炮弹类型与数据驱动

**目标：** 理解 CBC 炮弹的数据模型。

需要回答：
- 炮弹的属性（爆炸范围、伤害、穿深等）是硬编码还是数据驱动的？
- 是否有 JSON/DataPack 方式定义炮弹类型？
- CBC 附属（如 cbcmoreshells）如何注册新炮弹类型？
- 炮弹的投射物行为（弹道、重力、碰撞检测）在哪里实现？

### 3. 方块/实体交互

**目标：** 找到信标光柱和计时器的可行实现方式。

需要回答：
- CBC 中是否有类似的光柱/信号弹视觉效果？
- 原版信标光柱的渲染逻辑在哪里？能否在非信标方块位置生成光柱？
- 原版生物名牌的显示机制（在实体上方显示文字）？
- 是否有计时器/倒计时的 UI 显示机制？

### 4. 物品与交互系统

**目标：** 找到物品使用、投掷、状态切换的实现参考。

需要回答：
- CBC 中是否有手持物品右键交互的参考实现？
- 投掷物（类似雪球/末影珍珠）的实体基类是什么？
- 物品 NBT/组件系统如何存储状态（CD、已解锁战备等）？
- 战备指令输入系统（WASD序列匹配）的参考实现？

### 5. 兼容性设计

**目标：** 理解 CBC 的扩展点和兼容性约束。

需要回答：
- CBC 的炮弹系统是否有 EventBus/接口可以监听或扩展？
- 附属模组如何与 CBC 交互？走什么 API 层？
- 有哪些 CBC 的内部方法/类是不建议直接调用的（易引发兼容问题）？
- NeoForge 1.21.1 的注册系统（DeferredRegister 等）在 CBC 中的使用方式？
