# Prompt 01 - CBC 源码调研

## 角色

你是 CBC-Stratgems 的源码调研代理。你的任务是阅读 CreateBigCannons 源码，找出实现战略配备火力打击所需的真实代码路径。此阶段不要编写 CBC-Stratgems 的模组代码。

## 输入

- 玩法设想：`docs/stratagem-concept.md`
- CBC 上游仓库：https://github.com/Cannoneers-of-Create/CreateBigCannons
- 目标版本：NeoForge 1.21.1

如果本地没有 CBC 源码，可以将上游仓库克隆到 `.external/CreateBigCannons`，或使用用户已提供的本地路径。调研记录中写明实际使用的 CBC 提交、分支或 tag。

## 输出

创建或更新：

```text
research/cbc-source-map.md
```

该文件只记录源码事实、证据和待验证问题，不做最终架构决策。

## 调研纪律

- 使用 `rg` / `rg --files` 优先搜索源码。
- 每个结论都必须给出文件路径、类名、方法或字段名、行号范围。
- 如果只能推断，必须标记为“推断”，并说明缺少什么验证。
- 如果没找到直接实现，写“未找到”，不要用猜测填空。
- 不修改 CBC 源码。
- 不开始 CBC-Stratgems 实现代码。

## 必查模块

### 1. 炮弹生成与发射链路

回答以下问题：

- CBC 中炮弹实体的基类和主要子类是什么？
- 炮弹实体如何实例化？
- 发射或生成炮弹需要哪些参数：实体类型、坐标、朝向、速度、所有者、伤害、引信、弹药数据等。
- 是否存在可在任意位置生成并发射炮弹的现成 API。
- 如果没有现成 API，CBC 自身通过炮管发射时走了哪些类和方法。
- 散布、随机偏移、初速误差或精度相关逻辑在哪里实现。

### 2. 炮弹类型与数据模型

回答以下问题：

- 炮弹属性是硬编码、注册项驱动、JSON/DataPack 驱动，还是混合方式。
- 爆炸范围、伤害、穿深、引信、重力、阻力、碰撞行为分别在哪里定义。
- CBC 本体有哪些炮弹类型。
- 自定义炮弹通过什么注册路径加入。
- CBC 附属模组理论上如何注册新炮弹，并被 CBC 本体识别。
- 弹道 tick、碰撞检测、爆炸或命中逻辑在哪里实现。

### 3. 视觉效果与倒计时可行性

回答以下问题：

- CBC 是否已有类似信号弹、标记光束、目标指示器、倒计时提示或空袭提示。
- 原版信标光柱的渲染逻辑在 Minecraft/NeoForge 侧如何调用。
- 能否在非信标方块位置复用或模拟信标光柱。
- 实体名牌或自定义文字渲染的可用路径是什么。
- 如果图标和倒计时要显示在世界中，CBC 或 Minecraft 是否有可复用实体、渲染器或 HUD 参考。

### 4. 物品、投掷物与输入交互

回答以下问题：

- CBC 中有哪些右键使用、长按使用、持续使用或客户端 UI 打开的物品实现。
- 类似雪球、末影珍珠的投掷物基类在目标版本中是什么。
- CBC 是否有自定义投掷实体可参考。
- Minecraft 1.21.1 / NeoForge 中 ItemStack 数据组件如何用于存储状态。
- CBC 是否已有物品冷却、炮弹配置、物品状态切换或 NBT/组件迁移参考。
- WASD 指令输入在 CBC 中是否有参考；若没有，标记“未找到”并给出 Minecraft 客户端输入侧可查方向。

### 5. 兼容性与注册系统

回答以下问题：

- CBC 公开 API、接口、注册对象或 EventBus 扩展点有哪些。
- CBC 附属模组通常通过哪些包、接口、注册项与 CBC 交互。
- 哪些类看起来是内部实现，不适合作为长期兼容依赖。
- NeoForge 1.21.1 的 DeferredRegister 或等价注册方式在 CBC 中如何使用。
- 如果要自动兼容附属炮弹，最稳定的数据入口是什么。
